What is this?

This is a web app for visualising spatial data spatially and as humans experience it. It can simultanously display cartograms and geograhic maps.

A cartogram is a map where the size of each cell is proportional to a variable (e.g. population) rather than its geographic area.

Here, we have a population cartogram where the cells have been paired with physical locations using a matching algorithm such that each cell has roughly equal population (around 150k) while preserving the spatial coherence of the locations as far as possible.

Each cell is coloured according to the data displayed on the right hand side, aggregated by the weighted mean of the hexagonal cells according to the contribution of each hexagon to the total population of the cartogram cell.

All that is a complicated way of saying that the left hand side of this page displays data as experienced by people, and the right hand side displays the same data as experienced by land.

The mapping between the cartogram and geographic map can be counter-intuitive, but you can see exactly which geographic cells contribute to which cartogram cells, and vice versa, by clicking on them.

The original cartogram was created by Max Roser for Our World in Data; our routines for matching between the cartogram and geographic locations can be found on GitHub alongside the source code for this application.

Both the cartogram matching and this app itself were made by bovine3dom, who can usually be found hanging out on Mastodon.
